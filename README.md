OWASP ZAP Manual Scan Lab

Objective

The objective of this lab was to perform a manual web application security assessment using OWASP ZAP against the OWASP Juice Shop application. The assessment included discovering application resources, identifying security vulnerabilities, analyzing potential risks, and documenting remediation recommendations using industry-standard web application security testing techniques.

Tools Used

OWASP ZAP 2.17.0 – Web Application Security Scanner
OWASP Juice Shop – Vulnerable Web Application
Kali Linux 2026.x – Penetration Testing Operating System
Oracle VirtualBox – Virtualization Platform
Mozilla Firefox – Web Browser

Scan Methodology

The following methodology was used during the assessment:

Started the OWASP Juice Shop application.
Configured OWASP ZAP as the intercepting proxy.
Manually explored the application using Firefox.
Executed a Spider Scan to discover application pages and resources.
Performed an Active Scan to identify security vulnerabilities.
Reviewed and analyzed alerts generated by OWASP ZAP.
Documented findings and remediation recommendations.

Key Findings

The security assessment identified multiple vulnerabilities within the application, including:

SQL Injection
Missing Content Security Policy (CSP)
Missing Anti-Clickjacking Header
Cross-Domain Misconfiguration
Session ID in URL Rewrite
Vulnerable JavaScript Library
Application Error Disclosure
Cross-Domain JavaScript Source File Inclusion
Private IP Disclosure

These findings demonstrate common web application security weaknesses that attackers may exploit if left unaddressed.

Remediation Recommendations

To improve the application's security posture, the following recommendations are suggested:
Implement parameterized SQL queries to prevent SQL Injection.
Validate and sanitize all user input.
Configure a strong Content Security Policy (CSP).
Add security headers such as:
X-Frame-Options
X-Content-Type-Options
Strict-Transport-Security
Update vulnerable JavaScript libraries to supported versions.
Remove sensitive information from application error messages.
Secure session management and avoid exposing session identifiers in URLs.
Perform regular vulnerability assessments and penetration testing.

Conclusion

This lab demonstrated the practical use of OWASP ZAP to perform a manual security assessment of a web application. Through spidering, active scanning, and vulnerability analysis, several common web application security issues were identified. The exercise strengthened skills in vulnerability assessment, web application security testing, risk analysis, and remediation planning while reinforcing the importance of secure software development practices
